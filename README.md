# Pixel Quiz Quest - Retro Pixel Art Quiz Game

This is a React (Vite) based pixel art style quiz game. The quiz questions and score records are fully integrated with Google Sheets and Google Apps Script (GAS).

## 🚀 Installation & Running the Frontend Project

1. **Install Dependencies**
   Ensure Node.js is installed on your computer, then run the following in the project root directory:
   ```bash
   npm install
   ```
2. **Start the Development Server**
   ```bash
   npm run dev
   ```
   Once the server starts, open your browser and navigate to `http://localhost:5173`.

## 📊 Google Sheets Setup

You need to prepare a Google Spreadsheet and create **TWO** worksheets in it. The names must match exactly:

### First Sheet: `題目` (Questions)
Make sure the first row (Row 1) is a header row containing the following columns (order doesn't matter, but names must match exactly):
- `題號` (Question ID)
- `題目` (Question)
- `A`
- `B`
- `C`
- `D`
- `解答` (Answer)

### Second Sheet: `回答` (Answers)
This sheet is used to record player scores. The first row will be automatically generated by GAS, but you can pre-create the headers if you prefer:
- `ID`
- `闖關次數` (Attempts)
- `總分` (Total Score)
- `最高分` (Highest Score)
- `第一次通關分數` (Score of First Clear)
- `花了幾次通關` (Attempts Taken to Clear)
- `最近遊玩時間` (Last Played Time)

## ⚙️ Google Apps Script (GAS) Deployment Guide

1. In your Google Spreadsheet, click **"Extensions" -> "Apps Script"** from the top menu.
2. **Copy and paste** the entire code from `backend/gas.js` in your project into the opened `Code.gs` (overwrite all default code).
3. Click the **"Save project"** icon (the floppy disk).
4. Click **"Deploy" -> "New deployment"** in the top right corner.
5. Click the gear icon next to "Select type" and choose **"Web app"**.
6. Set the following configuration:
   - Description: Anything (e.g., Pixel Game API)
   - Execute as: **Me (your Google account)**
   - Who has access: **Anyone**
7. Click **"Deploy"**. If this is your first time, the system will ask to "Authorize access", please allow it and sign in with your Google account.
8. Once deployed, you will receive a **"Web app URL"** (ending in `/exec`). Please **copy** this URL.

## 🔧 Environment Variables (.env)

Open the `.env` file in the project root directory and configure the following variables:

```env
# Replace the URL below with the GAS Web app URL you just copied
VITE_GOOGLE_APP_SCRIPT_URL=https://script.google.com/macros/s/YOUR_URL_HERE/exec

# Passing threshold (How many questions to answer correctly to pass)
VITE_PASS_THRESHOLD=3

# Number of questions to be randomly drawn from the pool each game
VITE_QUESTION_COUNT=5
```

## 🌐 GitHub Pages Automatic Deployment

This project includes a pre-configured GitHub Actions workflow (`.github/workflows/deploy.yml`) to automatically deploy your game to GitHub Pages whenever you push code to the `main` or `master` branch.

### Deployment Steps:

1. **Push your code to GitHub**
   Initialize a git repository, commit all files, and push to a new GitHub repository.
2. **Setup Repository Secrets for Environment Variables**
   Because environment variables are needed during the build process to connect to your GAS backend, you **must** add them to your GitHub repository secrets:
   - Go to your repository on GitHub.
   - Click **Settings** > **Secrets and variables** > **Actions**.
   - Click **New repository secret** and add the following three secrets one by one (using the values from your `.env` file):
     - `VITE_GOOGLE_APP_SCRIPT_URL`
     - `VITE_PASS_THRESHOLD`
     - `VITE_QUESTION_COUNT`
3. **Enable GitHub Pages settings**
   - In your repository, go to **Settings** > **Pages**.
   - Under **Build and deployment**, change the **Source** dropdown to **GitHub Actions**.
4. **Trigger Deployment**
   Once you've pushed your code and configured the secrets and settings above, GitHub Actions will automatically start building and deploying your project. You can check the progress in the **Actions** tab of your repository.
   Once the workflow completes successfully, your game will be live for everyone to play!
